# BlanketOps Environments Contract

## Overview

`BlanketOps Environments Contract` defines the canonical contracts for BlanketOps Environments.

This repository contains versioned protobuf schemas that describe:

- environment intent
- build and deployment workflows
- routing and service units
- external events and sources
- shared primitives used across all API groups

These contracts are the source of truth for how BlanketOps components communicate and reason about environments.

Each API group:

## Versioning Model

This repository follows semantic versioning at the contract level. The full promotion and compatibility rules are defined in [API_VERSIONING_POLICY.md](API_VERSIONING_POLICY.md).

### API versions

- `v1alpha1` вЂ” experimental, breaking changes allowed
- `v1beta1` вЂ” feature complete, limited breaking changes
- `v1` вЂ” stable, backward-compatible

Once an API reaches `v1`:

- fields are not removed
- semantics are preserved
- evolution happens via additive changes only

Breaking changes require a new API version.

Release history is tracked in [CHANGELOG.md](CHANGELOG.md).

## Code Generation

This repository uses [Buf](https://buf.build) for protobuf generation and consistency.

Relevant files:

- `buf.yaml` вЂ” module configuration and lint rules
- `buf.gen.yaml` вЂ” generation configuration (Go, Go gRPC, and TypeScript via protobuf-es)

Generated code is committed intentionally to:

- ensure reproducible builds
- simplify downstream consumption
- avoid forcing consumers to run codegen

## Consuming the Contracts

Go consumers import the module directly:

```sh
go get github.com/BlanketOps/environments-contract
```

```go
import envv1 "github.com/BlanketOps/environments-contract/blanketops/environments/v1"
```

TypeScript consumers use the committed `_pb.ts` sources generated by protobuf-es v2.

## Tooling

Common tasks are exposed via [Mage](https://magefile.org). Run:

```sh
mage help
```

Available targets include:

- `mage lint` вЂ” validate protobuf definitions against Buf lint rules
- `mage gen` вЂ” regenerate Go and TypeScript bindings
- `mage verify` вЂ” ensure committed generated code matches the schemas
- `mage bundle` вЂ” produce versioned contract archives
- `mage publish` вЂ” publish release artifacts

## Consumers

This repository is consumed by:

- BlanketOps operators and controllers
- Internal policy and resolution engines
- Tooling that requires a stable environment contract
